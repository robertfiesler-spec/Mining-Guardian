# Components: Buttons, Forms & Copywriting

## Buttons

### Button Hierarchy

| Type | Usage | Styling |
|------|-------|---------|
| Primary | Most important action (ONE per view) | Filled, brand color |
| Secondary | Supporting actions | Outlined or muted fill |
| Tertiary | Least important actions | Text-only, minimal styling |

### Button Rules

- ✅ Use a single primary button per view
- ✅ Left-align buttons (especially for screen magnifier users)
- ✅ Ensure button text describes the action ("Save changes", not "Submit")
- ✅ Minimum touch target: 44×44px (WCAG 2.5.5 AAA)
- ✅ Add confirmation dialogs to destructive actions
- ✅ Balance icon + text pairs visually
- ❌ Avoid disabled buttons - explain why action isn't available instead

## Forms

### Structure

- ✅ Stack forms in a single column
- ✅ Minimize the number of fields
- ✅ Match field width to expected input length
- ✅ Break long forms into steps
- ✅ Group related fields under headings

### Labels & Hints

- ✅ Mark optional fields (not required ones)
- ✅ Display hints above form fields
- ✅ Keep labels close to their fields
- ❌ Never use placeholder text as labels

### Input Selection

| Options | Use |
|---------|-----|
| 2 options | Checkbox or toggle |
| 3-5 options | Radio buttons |
| 6+ options | Dropdown or autocomplete |
| Numeric | Stepper control |

### Validation

- ✅ Validate on submit (not inline as user types)
- ✅ Use multiple visual cues for errors: color + icon + border + background
- ✅ Write clear, specific error messages
- ✅ Position errors near the relevant field

### Form Field Styling

- ✅ Use conventional form field styles
- ✅ Ensure field borders have 3:1 minimum contrast
- ✅ Make focus states clearly visible

## Copywriting

### General Principles

- ✅ Be concise - remove unnecessary words
- ✅ Use sentence case (not Title Case)
- ✅ Use plain, simple language
- ✅ Front-load text - put important info first
- ✅ Use numerals for numbers (5, not "five")

### Labels & Links

- ✅ Use vocabulary consistently throughout
- ✅ Make link text describe the destination (not "click here")
- ❌ Avoid "my" in form labels - use "Your" or nothing
- ❌ Avoid full stops in headings and buttons
- ❌ Limit abbreviations and acronyms
- ❌ Limit UPPERCASE text

### Text Formatting

- ✅ Break up content with descriptive headings
- ✅ Use bullets only for scannable lists
- ✅ Ensure similar UI elements have similar text length
